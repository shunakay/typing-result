# ⌨️ タイピング成績ダッシュボード

タイピング練習（e-typing 等）の成績スクリーンショットを AI で読み取り、
Google スプレッドシートに記録・蓄積して、ダッシュボードで可視化する静的 Web アプリです。

## 主な機能

| 機能 | 説明 |
|------|------|
| 📊 ダッシュボード | 得点推移・正タイプ率・文字数・誤タイプ数のグラフを全員に公開 |
| 🤖 AI 読み取り | スクリーンショットを Gemini API が自動解析してフォームに流し込み |
| ✏️ 手動入力 | 画像なしでも記録可能。正タイプ率・誤タイプ率は自動計算 |
| 🎯 目標達成率 | 設定した目標値に対するプログレスバーを表示 |
| 📋 履歴テーブル | 全記録を一覧表示・CSV エクスポート |
| 🔒 アクセス制御 | 閲覧は誰でも可・記録は GAS URL を知る所有者のみ |

## セットアップ手順

### Step 1: スプレッドシートを作成する

1. [Google スプレッドシート](https://sheets.google.com) で新規スプレッドシートを作成
2. シート名を `typing_records` に変更（タブをダブルクリックして編集）
3. **1行目**に以下のヘッダーをコピーして貼り付け

```
日付	入力方式	制限時間(分)	得点	入力文字数	正タイプ数	誤タイプ数	正タイプ率(%)	誤タイプ率(%)	メモ
```

> **ポイント**: タブ区切りで貼り付けると A〜J 列に自動で入ります。

---

### Step 2: Google Apps Script をデプロイする

1. スプレッドシートのメニュー → **拡張機能** → **Apps Script** を開く
2. エディタに表示されているコードを全削除し、`gas/Code.gs` の内容を貼り付け
3. 保存（Ctrl+S / ⌘S）
4. **デプロイ** → **新しいデプロイ** をクリック
5. 種類の選択 → ⚙️ アイコン → **ウェブアプリ** を選択
6. 以下のように設定してデプロイ:

   | 項目 | 設定値 |
   |------|--------|
   | 説明 | 任意（例: typing dashboard v1） |
   | 次のユーザーとして実行 | **自分** |
   | アクセスできるユーザー | **全員** |

7. **デプロイ** ボタンを押し、Google アカウントで承認
8. 発行された **ウェブアプリ URL** をコピーして控えておく

   ```
   例: https://script.google.com/macros/s/AKfy.../exec
   ```

---

### Step 3: Gemini API キーを取得する

1. [Google AI Studio](https://aistudio.google.com/) にアクセス
2. **Get API key** → **Create API key** でキーを発行
3. **セキュリティ設定（必須）**: API キーの制限を設定する
   - キーの ✏️ 編集 → **アプリケーションの制限** → **HTTP リファラー（ウェブサイト）**
   - 許可するリファラーに GitHub Pages の URL を追加:
     ```
     https://<your-username>.github.io/*
     ```
   これにより、自分のサイト以外からのキー悪用を防げます。

---

### Step 4: `js/config.js` を設定する

`js/config.js` を開き、以下の 2 箇所を書き換えます:

```js
const CONFIG = {
  // ↓ Step 3 で取得した Gemini API キー
  GEMINI_API_KEY: 'AIzaSy...',

  // ↓ Step 2 で取得した GAS Web App URL（読み取り用・公開 OK）
  GAS_READ_URL: 'https://script.google.com/macros/s/.../exec',

  // 目標値（お好みで変更可）
  TARGETS: {
    score: 5000,
    correctRate: 99.0,
    charCount: 500,
    missCount: 5,
  },

  INPUT_TYPES: ['英字', '日本語', 'かな', 'その他'],
  ...
};
```

---

### Step 5: GitHub リポジトリを作成して push する

```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/<your-username>/<repo-name>.git
git push -u origin main
```

> `config.js` には API キーが含まれます。  
> `.gitignore` には記載していませんが（GitHub Pages に必要なため）、  
> **必ず Google AI Studio でリファラー制限**を設定してください。

---

### Step 6: GitHub Pages を有効化する

1. GitHub リポジトリの **Settings** タブを開く
2. 左メニュー → **Pages**
3. Source: **Deploy from a branch**
4. Branch: `main` / `/ (root)` → **Save**
5. 数分後に `https://<your-username>.github.io/<repo-name>/` でアクセス可能

---

### Step 7: 記録モードを設定する（所有者のみ）

1. 公開された GitHub Pages の URL にアクセス
2. 右上の **⚙️** ボタンをクリック
3. **GAS Web App URL** 欄に Step 2 の URL を貼り付けて **保存**
4. ページがリロードされ、🔓 記録モードが有効に
5. 以降はスクリーンショットをアップロードするだけで記録可能

---

## 使い方

### 閲覧者向け

URL にアクセスするだけで、すべてのグラフと履歴を閲覧できます。  
認証不要・ログイン不要です。

### 記録者（所有者）向け

**画像で記録する（推奨）**

1. タイピングサイトの成績画面をスクリーンショット
2. ダッシュボード上部のドロップゾーンに画像をドラッグ&ドロップ
3. AI が自動解析してフォームに入力（数秒）
4. 内容を確認・修正して「スプレッドシートに保存」

**手動で記録する**

1. フォームの各項目を手入力
2. 正タイプ数・誤タイプ数を入力すると正タイプ率・誤タイプ率が自動計算
3. 「スプレッドシートに保存」

**記録を編集・削除する**

- 履歴テーブルの「編集」「削除」ボタンが記録モード時のみ表示されます

## カスタマイズ

### 目標値を変更する

`js/config.js` の `TARGETS` を編集します:

```js
TARGETS: {
  score: 6000,       // 得点目標
  correctRate: 99.5, // 正タイプ率目標（%）
  charCount: 600,    // 入力文字数目標
  missCount: 3,      // 誤タイプ数目標（以下が目標）
},
```

### 入力方式を追加する

```js
INPUT_TYPES: ['英字', '日本語', 'かな', '英語長文', 'プログラム'],
```

## アーキテクチャ

```
GitHub Pages（静的ファイル）
  ├── index.html
  ├── js/
  │   ├── config.js     ─── APIキー・目標値の設定
  │   ├── gas-client.js ─── GAS Web App との通信
  │   ├── ocr.js        ─── Gemini API でのOCR処理
  │   ├── dashboard.js  ─── Chart.js グラフ描画
  │   ├── recorder.js   ─── 記録モードUI・フォーム制御
  │   └── app.js        ─── 初期化・統合
  └── gas/
      └── Code.gs       ─── GAS にコピーするコード

外部サービス:
  ├── Gemini API (gemini-2.0-flash) ─── 画像OCR
  └── Google Apps Script             ─── スプレッドシートの読み書き Proxy
```
